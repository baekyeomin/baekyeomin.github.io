## 2. 문제 분석
#### 🐾 생각 흐름 (1차시도-시간초과) 
처음에는 그냥 재귀 그댜로, dp 그대로 함수 돌려서 피보나치를 계산하면서 코드1과 코드2 실행 횟수를 세면 될거라고 생각했고, VS 에서 돌렸을 때는 또 잘 돌아갔는데 시간초과가 났다.. ㅜㅜ

```python
n = int(input())
recur_cnt = 0
dp_cnt = 0

def fib(n) :
    if n == 1 or n == 2 :
        global recur_cnt
        recur_cnt += 1
        return 1 #코드1
    else :
        return (fib(n - 1) + fib(n - 2))


def fibonacci(n) :
    f = [1,1]
    for i in range(3,n+1) :
        global dp_cnt
        dp_cnt += 1
        f.append(f[i - 2] + f[i - 3])  # 코드2
    return f[n-1]

fib(n) 
fibonacci(n)

print(recur_cnt , dp_cnt)
```
<span style="color:red;"><u>재귀 그대로 돌려서 횟수를 세면 시간초과가 뜬다!</u></span> <br> 이게 왜 "수학"으로 분류되어있나 했더니 출력결과를 보고 있다보니 수학적으로 계산해서 출력하는 문제 아닐까하는 생각이 들었다. 


#### 🐾 생각 흐름 (2차시도-맞았습니다) 
1. recur_cnt = 코드1이 실행되는 횟수 , dp_cnt = 코드2가 실행되는 횟수
2. recur_cnt는 피보나치수 (F(n))
- fib(n)에서 return 1 (코드1)이 실행되는 횟수는 F(n)
3. dp_cnt는 n-2
- for i = 3..n에서 코드2가 딱 한 번씩 실행되니까

```python
n = int(input())
recur_cnt = 0
dp_cnt = 0

f1 = 1
f2 = 1
for _ in range(3,n+1) :
    f1, f2 = f2, f1+f2
    
recur_cnt = f2
dp_cnt = n-2

print(recur_cnt , dp_cnt)
```
