## 과적합과 과소적합
데이터가 적으면 단순한 모델이 최적이고, <br>
데이터가 많으면 복잡한 모델이 더 나은 성능을 보인다. <br>
적절한 모델 복잡도는 데이터의 수에 따라 달라진다. 

## 정규화
일반적으로는 비용함수에 정규화항을 추가해서 패널티를 부여하는 방식. <br>

### dropout
훈련 과정 중 무작위로 일부 뉴런을 drop하여 학습을 수행하는 방법 <br>
**dropout은 앙상블과 유사함**
- 매 훈련 단계마다 다양한 subset을 학습하면서 서로 다른 특성을 학습하기에 전체 모델은 다양한 학습된 특성을 통합해서 더 강력해진다는 점에서 유사하다. 
  
### 데이터 증강
여러 패치를 사용하여 각기 다른 이미지 변형을 반영하도록하여 일반화 성능을 개선하는 방식 <br>
예 : Alexnet에서 데이터 증강 : <br>
256*256 크기의 원본 이미지를 10개의 224*224 패치를 통해 10개의 소프트맥스 출력의 평균을 기반으로 예측을 수행하도록함

### train set & validation set
일반적으로 훈련데이터의 80%는 훈련에 사용하고, 20%는 검증에 사용한다.
<br>

## 적절한 훈련 및 평가 방법
code 작성 -> train -> validation (반복)<br>
-> test 

<span style="color: blue">하이퍼파라미터 조정 및 early stop 을 위해 별도의 검증 세트 두는 것이다. </span> <br> <br>
train,validation set이 작을 때, 추정된 오류에 대한 통계적 불확실성이 초래될 수 있기에 k-겹교차검증(k-fold cross validation)을 사용한다. 

### k-겹교차검증(k-fold cross validation)
- 전체 데이터를 k개의 겹치지 않는 하위 집합으로 나눈다
- k번의 라운드 로빈 실험 수행 : 데이터의(k-1)/k로 훈련하고, 1/k로 테스트하기를 반복
- k 번의 실험에서 얻은 오류 평균내어 테스트 오류룰 추정
  <br> <br>
1. (작은) 훈련 데이터셋을 여러 개의 폴드(folds)로 나눈다.
2. 각 하이퍼파라미터 조합에 대해 교차 검증을 수행하고 평균 오류를 측정한다.
3. 평균 오류 측면에서 가장 좋은 조합을 선택한다.
4. 선택한 하이퍼파라미터 조합을 사용하여 전체 훈련 데이터셋으로 모델을 다시 훈련시킨다.<br> <br>


위에 hat ^ 씌워서 표기하는, 특정 파라미터의 값을 어떤 수치로 추정하는거 = 점추정 

### Bias-Variance 트레이드오프
Bias가 높은 추정기 : <br>
- 복잡한 데이터에 비해 지나치게 단순한 모델이면 데이터 패턴을 충분히 학습하지 못하게됨<br><br>
  
Variance가 높은 추정기 : <br>
- 데이터의 모든 세부사항을 학습하려고 하면서 모델이 데이터의 노이즈까지 학습함

<span style="background-color: #fff3cd">모델의 capacity가 증가할 수록 bias는 감소하고 variance는 증가하는 경향</span>이 있고 그래프를 그리면 아래와 같다. 

![dl_chp4_39p](/assets/images/dl_chp4_39p.png) 

[트레이드 오프 조정하는 방법] <br>
1. 교차 검증 (Cross Validation)<br>
데이터를 여러번 나누어 모델을 평가 -> 편향과 분산의 균형 평가

2. MSE <br>
MSE는 Bias와 Variance 동시에 고려하는 지표<br>($MSE = Bias^2 + Variance + Noise = \mathbb{E}[(\hat{\theta}_m - \theta)^2] = Bias(\hat{\theta}_m)^2 + Var(\hat{\theta}_m)$)


## 경사하강법
미분가능한 공간에서 local minimum 찾기 위해 사용되는 최적화 알고리즘으로, 매 반복마다 파라미터를 가장 가파른 하강 방향(기울기값의 음수값)으로 업데이트하여 최솟값을 찾음

![dl_chp4_44p](/assets/images/dl_chp4_44p.png) 

### 1. Batch Gradient Descent
- 전체 훈련 데이터에 대해 기울기 계산
- 데이터가 많으면 계산 비용이 큼
- 손실 함수가 매끄러워 안정적이지만 수렴 속도가 느림

### 2. Stochastic Gradient Descent (SGD)
- 무작위로 샘플링된 **단일 데이터**로 기울기 계산
- 계산량이 작음
- 노이즈로 인해 최적점 근처에서 진동 가능
- 가장 많이 사용됨
  
### 3. Mini-batch Gradient Descent
- 무작위로 샘플링된 **소규모 데이터(batch)** 로 기울기 계산
- 일반적으로 **10~1000개의 데이터 사용**
- 계산 효율성과 수렴 속도의 균형
- 딥러닝에서 가장 많이 사용
